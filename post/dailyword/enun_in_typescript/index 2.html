Enum_in_TypeScript

@Herr Du · Oct 17, 2019 · 3 min read

在类似于C、C#和Java这些强类型的语言里都会有一个数据类型是 —— enumerations（enums是简称）。不过在JS里面似乎并没有这个数据类型。庆幸的是，在MS开发的TypeScript中提供了这个类似，我们可以在TS2.4以后的版本上使用。Enums可以让我们声明和定义一个常量数据集来阐述某一个概念下的相关的数值和字符含义。

先看一下Enum的语法

enum Color {
    Red,
    Green,
    Blue
}
var col = Color.Red;
col = 0; // Effectively same as Color.Red

在看一下 TS 是如何编译 enum 的

enum Tristate {
    False,
    True,
    Unknown
}

被TS生成的代码

var Tristate;
(function (Tristate) {
    Tristate[Tristate["False"] = 0] = "False";
    Tristate[Tristate["True"] = 1] = "True";
    Tristate[Tristate["Unknown"] = 2] = "Unknown";
})(Tristate || (Tristate = {}));

我们会注意到 Tristate[Tristate["False"] = 0] = "False"这一段中的Tristate["False"] = 0。这意味着我们可以把 Tristate 一个字符串类型的 enum 转化成一个数字类型的enum，反之亦然。这在 enum 里面称之为反向枚举（Reverse mappings）

enum Tristate {
    False,
    True,
    Unknown
}
console.log(Tristate[0]); // "False"
console.log(Tristate["False"]); // 0
console.log(Tristate[Tristate.False]); // "False" because `Tristate.False == 0`

数字枚举的默认值与自增

自增。 当我们给Up赋值1的时候，Down、Left、Right的值会自增为2,3,4.

enum Direction {
    Up = 1,
    Down,
    Left,
    Right
}

不连续的自增

enum Weekend {
	Friday = 1,
	Saturday = 13,
	Sunday,
}


var Weekend;
(function (Weekend) {
    Weekend[Weekend["Friday"] = 1] = "Friday";
    Weekend[Weekend["Saturday"] = 13] = "Saturday";
    Weekend[Weekend["Sunday"] = 14] = "Sunday";
})(Weekend || (Weekend = {}));

默认值。如果我们不赋值的话，则 Up、Down、Left、Right 的值会定义为 0、1 、2、3 。如下：

enum Direction {
    Up,
    Down,
    Left,
    Right,
}

字符串枚举

字符串枚举有利于序列化

enum Direction {
    Up = "UP",
    Down = "DOWN",
    Left = "LEFT",
    Right = "RIGHT",
}

异构枚举

从技术角度看，枚举可以混合字符串和数字类型的成员。不过我们一般不会这么做

enum BooleanLikeHeterogeneousEnum {
    No = 0,
    Yes = "YES",
}

计算成员的枚举

// All enum members in 'E1' and 'E2' are constant.

enum E1 { X, Y, Z }

enum E2 {
    A = 1, B, C
}

enum FileAccess {
    // constant members
    None,
    Read    = 1 << 1,
    Write   = 1 << 2,
    ReadWrite  = Read | Write,
    // computed member
    G = "123".length
}


enum Weekend {
  Friday = 1,
  Saturday = getDate('TGIF'),
  Sunday = Saturday * 40
}

function getDate(day : string): number {
    if (day === 'TGIF') {
        return 3;
    }
}
Weekend.Saturday; // returns 3
Weekend.Sunday; // returns 120

联合枚举与枚举成员的类型

下面的代码中，Circle 的 kind 属性是联合了 ShapeKind，并且使 ShapeKind 的 Circle成员变成了一个成员类型。

enum ShapeKind {
    Circle,
    Square,
}

interface Circle {
    kind: ShapeKind.Circle;
    radius: number;
}

interface Square {
    kind: ShapeKind.Square;
    sideLength: number;
}

let c: Circle = {
    kind: ShapeKind.Square,
    //    ~~~~~~~~~~~~~~~~ Error!
    radius: 100,
}

下面的例子中， 如果判断了 x !== E.Foo 则 x 的值只能是 E， 则无需再判断 x !== E.Bar

enum E {
    Foo,
    Bar,
}

function f(x: E) {
    if (x !== E.Foo || x !== E.Bar) {
        //             ~~~~~~~~~~~
        // Error! Operator '!==' cannot be applied to types 'E.Foo' and 'E.Bar'.
    }
}

运行时的枚举

函数 f 在调用 E 时没有报错，是因为 E 中是有 X 这个成员的

enum E {
    X, Y, Z
}

function f(obj: { X: number }) {
    return obj.X;
}

// Works, since 'E' has a property named 'X' which is a number.
f(E);

const 枚举

有时为了提高性能，我们可以使用常量枚举,这样在编译的时候直接把枚举成员的值赋值给引用的变量

enum Weekend {
  Friday = 1,
  Saturday,
  Sunday
}
var day = Weekend.Saturday;

编译后

var Weekend;
(function (Weekend) {
    Weekend[Weekend["Friday"] = 1] = "Friday";
    Weekend[Weekend["Saturday"] = 2] = "Saturday";
    Weekend[Weekend["Sunday"] = 3] = "Sunday";
})(Weekend || (Weekend = {}));
var day = Weekend.Saturday;



const enum Weekend {
	Friday = 1,
	Saturday,
	Sunday,
}
var day = Weekend.Saturday;

编译后

var day = 2 /* Saturday */;
