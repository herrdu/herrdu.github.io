每日一词：Event

@Herr Du · Jul 14, 2019 · 2 min read

大概每个写过 HTML 的人都会知道 Event 接口，它为 DOM 的处理函数提供了上下文的描述。一般地说，事件的处理函数会把实现了 Event 接口的对象作为第一个参数传递过去。不过大多数 DOM 事件处理函数传递的上下文对象中，还会传递一些额外的信息。如：MouseEvent 的处理函数种会有 clientX 和 clientY 这些信息。

而可以调用事件处理函数的可以是用户操作设备引起的（如：鼠标和键盘的操作），也可以是 API Hook 引起的（如：音视频的 process，pause ），还有时可以是 EventTarget.dispatchEvent 引起的。

下面看一下 Event 模型的 IDL

IDL

[Constructor(DOMString type, optional EventInit eventInitDict),
 Exposed=(Window,Worker,AudioWorklet)]
interface Event {
  readonly attribute DOMString type;
  readonly attribute EventTarget? target;
  readonly attribute EventTarget? srcElement; // historical
  readonly attribute EventTarget? currentTarget;
  sequence<EventTarget> composedPath();

  const unsigned short NONE = 0;
  const unsigned short CAPTURING_PHASE = 1;
  const unsigned short AT_TARGET = 2;
  const unsigned short BUBBLING_PHASE = 3;
  readonly attribute unsigned short eventPhase;

  void stopPropagation();
           attribute boolean cancelBubble; // historical alias of .stopPropagation
  void stopImmediatePropagation();

  readonly attribute boolean bubbles;
  readonly attribute boolean cancelable;
           attribute boolean returnValue;  // historical
  void preventDefault();
  readonly attribute boolean defaultPrevented;
  readonly attribute boolean composed;

  [Unforgeable] readonly attribute boolean isTrusted;
  readonly attribute DOMHighResTimeStamp timeStamp;

  void initEvent(DOMString type, optional boolean bubbles = false, optional boolean cancelable = false); // historical
};

dictionary EventInit {
  boolean bubbles = false;
  boolean cancelable = false;
  boolean composed = false;
};

属性

  • Event.type
    事件的类型，具体类型看参考 1

  • Event.target
    返回触发 event（事件）的 dom 对象

  • Event.srcElement
    Microsoft Internet Explorer 种的遗留产物， 现在推荐使用 Event.target

  • Event.currentTarget
    返回使用 addEventListern 监听的事件

  • Event.eventPhase
    返回事件在哪个发生阶段，共有四个阶段：NONE, CAPTURING_PHASE, AT_TARGET, and BUBBLING_PHASE。

  • Event.bubbles
    表示该事件是否在 DOM 种冒泡

  • Event.cancelBubble
    Event.stopPropagation() 以前的别名。通过在一个事件处理程序返回前设置这个属性的值为真，来阻止事件冒泡。

  • Event.cancelable
    用来表示这个事件默认行为是否可以取消。如果 cancelable 的值为 false 的时候，调用 preventDefault() 将产生错误。

  • Event.composed
    感觉 event 初始化的内容返回 true 或者 false. True 表示触发事件的对象是一个 ShadowRoot node 且是 target 的 root；false 则表示相反的情况。

  • Event.returnValue
    旧版的 IE 的遗留产物，和 Event.preventDefault() 和 Event.defaultPrevented 有一样的作用。

  • Eevet.defaultPrevented
    表示了是否已经执行过了 event.preventDefault()

  • Event.isTrusted
    指明事件是否是由浏览器（当用户点击实例后）或者由脚本（使用事件的创建方法，例如 event.initEvent）启动

  • Event.timeStamp 表示从 UTC 1970 年 1 月 1 日都事件触发时的毫秒数

方法

  • composedPath()
    返回事件路径数组。如果 target 是影子根节点且 ShadowRoot.mode 是关闭的，则该路径不包括影子树中的节点。

  • stopPropagation()
    阻止事件冒泡

  • stopImmediatePropagation()
    如果有多个相同类型事件的事件监听函数绑定到同一个元素，当该类型的事件触发时，它们会按照被添加的顺序执行。如果其中某个监听函数执行了 event.stopImmediatePropagation() 方法，则当前元素剩下的监听函数将不会被执行。

  • preventDefault() 告诉 user agent，如果此事件没有被显式的处理，则它的默认动作也不要做。如：form > button 的默认事件可能会执行一个表单提交。如果执行了 preventDefault()，那么不再会有提交动作。
