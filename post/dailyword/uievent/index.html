每日一词：UIEvent

@Herr Du · Jul 17, 2019 · 1 min read

UIEvent 接口是从 Event 接口派生出来的。出行这个接口有两个原因，第一个就是为了设计一个允许事件监听器注册和能够在树形结构里描述事件流动的接口；第二个原因是在流量器里为现有的事件系统提供一个通用的子集。

DOM中事件的流动

一个应用可以通过 dispatchEvent() 来分发事件，event 的传播过程会按照 DOM 事件流规定进行传播。 下图显示了事件是如何在 DOM 树中进行传播的。

浏览器产生的事件对象会被分发到某一个具体的对象上面去，但是的分发开始前，会先确定好事件对象的传播路径（propagation path ）。

propagation path 是由 current event targets 组成的有序列表。它的最后的一个元素是发生事件的 event target，event target 紧邻的上一个元素是它的父节点，父节点之前的是它的祖先节点。

一旦确定了 propagation path，那么 event object 在上面的一个或多个阶段进行传播。有三个典型的传播阶段：

  1. capture phase（捕获阶段）
    事件有顶层 event target 传播的过程
  2. target phase（目标阶段）
    event object 到达 event target 上的阶段
  3. bubble phase（冒泡阶段）
    event object 由event target 传向 window 阶段

如果 event 的 bubbles 属性设置为 false 的时候，event 的传播过程就是跳过 bubble phase；如果调用 stopPropagation() 的话，剩下的所有的传递行为都会跳过。

IDL

[Constructor(DOMString type, optional UIEventInit eventInitDict), Exposed=Window]
interface UIEvent : Event {
  readonly attribute Window? view;
  readonly attribute long detail;
};

dictionary UIEventInit : EventInit {
  Window? view = null;
  long detail = 0;
};

构造方法

event = new UIEvent(typeArg [, UIEventInit])

派生的接口

  • FocusEvent
  • MouseEvent
  • WheelEvent
  • InputEvent
  • KeyboardEvent
  • CompositionEvent
